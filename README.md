# こうかとんマスターズ

## 実行環境の必要条件
* python >= 3.10
* pygame >= 2.1

## ゲームの概要
* ライフ制、PvPのカードゲーム
　ライフを削りきるもしくは相手の山札がなくなったら勝ち
* デッキ枚数20枚、5種4枚、使ったカードは墓地
* 参考

## ゲームの遊び方
* ランダムで先攻後攻が決まり、はじめの手札が3枚
* 毎ターンマナと手札が1つずつ増える（先攻1ターン目は手札が増えない）
* アタックすると指定の数のライフを削るかタップしているクリーチャーにアタックできる、出たターンアタック不可、アタック後にタップする


## ゲームの実装
### 共通基本機能
* 
* カードゲームで遊ぶことができるゲーム
* 参考URL：[ChatGPT](https://chatgpt.com/)

## ゲームの遊び方
* 二人のプレイヤーが一つの画面を使い、お互いの手札が見えた状態でプレイする。
* Lifeが０になると敗北、もしくはデッキが０枚になると敗北。
* 初期手札は３枚、使用可能最大マナは１。（いずれもターン経過で１ずつ増加する）
* ターンプレイヤーが自身の手札からコストが使用可能マナ以下になるようにカードをクリックした後、Use_Cardボタンをクリックするとカードが使用できる。
* フィールドにあるクリーチャーはそのクリーチャーをクリックした後、Attackボタンをクリックすると攻撃できる。
* フィールドにあるカードは出たターンはすぐに攻撃できない。（次のターンから攻撃可能）
* 攻撃したクリーチャーはタップする。（灰色になる）（次の自分のターンに元に戻る）
* 相手プレイヤーへの直接攻撃だけでなく、タップしているクリーチャーに向かって攻撃することもできる。攻撃カードを選択した後、Attackボタンを押す前に相手のクリーチャーを選択してからAttackボタンをクリックするとバトルができる。

## ゲームの実装
### 共通基本機能
* 共通基本機能はChatGPTで制作し、手動で手直ししたもの
* 背景画像と手札の使用、場のカードの表示
* プレイヤーへの直接攻撃
* ライフの初期値と減少
* カード初期設定
* ２０枚のデッキの設定
* 敗北判定

### 分担追加機能
* BGM追加（担当:山口）：戦闘中のBGMを追加した
* BGM変更（担当:山口）：ライフ3以下の時のBGMを変更した
* バトル処理関数（担当：藤田）：クリーチャー同士のバトルに関する関数
* 攻撃先選択（担当：藤田）：相手プレイヤー以外にも、タップ（横向きになっているカード）しているクリーチャーに攻撃し、バトルを仕掛けることができる機能
